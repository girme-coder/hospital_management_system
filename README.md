# Hospital Management System

## Project Overview

The Hospital Management System is a console-based application developed in C++ using MySQL and MySQL Connector/C++. It is designed to manage core hospital operations such as patient management, doctor management, appointment scheduling, and reporting.

This project helps medical offices and hospital administrators organize records, schedule appointments, and generate performance summaries. It solves the problem of manual data handling by providing a structured database-backed workflow for hospital administration.

## Features

- Add, view, and manage patients
- Add, view, and manage doctors
- Book appointments
- Validate appointment date and time for future-only scheduling
- Generate reports including:
  - Total number of patients
  - Total number of doctors
  - Total number of appointments
  - Doctor activity report with appointments per doctor

## Technologies Used

- C++ (C++17)
- MySQL Database
- MySQL Connector/C++

## Database Structure

The system uses a MySQL database with the following primary tables:

- `Patients`
  - Stores patient details such as name, contact information, and medical record identifiers.

- `Doctors`
  - Stores doctor details including name, specialty, and availability information.

- `Appointments`
  - Stores scheduled appointments, linking patients and doctors with date and time fields and status information.

## How to Compile and Run

Use the following command to build the application:

```bash
g++ main.cpp database.cpp patient.cpp doctor.cpp appointment.cpp utils.cpp report.cpp -o hms -std=c++17 -lmysqlcppconn
```

Then run the executable:

```bash
./hms
```

And make sure that you manually change your mysql name and password on `database.cpp` and create mysql table similar with `schema.sql`.

## Sample Features Explanation

- **Booking Appointment**: The system collects patient and doctor information, selects a future date and time, and inserts a new appointment record into the database.
- **Validation**: Appointment scheduling includes validation logic that ensures the selected date and time are in the future, preventing invalid or past bookings.
- **Reports**: Reports are generated by querying the database for summary values. The application calculates totals for patients, doctors, and appointments, and creates a doctor activity report showing appointment counts per doctor.
